# Claude Code — Полная база знаний (актуально на май 2026)

## ЧТО ТАКОЕ CLAUDE CODE

Claude Code — это CLI-инструмент (интерфейс командной строки) от Anthropic, который запускается прямо в терминале. Он читает ваш код, выполняет команды, редактирует файлы, работает с git, подключается к внешним сервисам через MCP и делегирует задачи специализированным субагентам.

**Где работает:**
- Терминал / командная строка
- VS Code (расширение)
- JetBrains IDE (расширение)
- Десктоп-приложение (Mac/Windows)
- Веб-приложение (claude.ai/code)

## УСТАНОВКА И ЗАПУСК

```bash
# Установка через npm
npm install -g @anthropic-ai/claude-code

# Проверка версии
claude --version

# Обновление
claude update

# Авторизация
claude auth login

# Запуск интерактивной сессии
claude

# Запуск с начальным запросом
claude "объясни структуру этого проекта"

# Неинтерактивный режим (выполнить и выйти)
claude -p "добавь типы TypeScript в этот файл"

# Продолжить последнюю сессию
claude -c
claude --continue

# Продолжить сессию по ID
claude -r <session-id>
```

## СКИЛЛЫ (SKILLS) — ПОДРОБНО

### КАК ОБЪЯСНИТЬ СКИЛЛ НОВИЧКУ (используй именно это при первом знакомстве)

Три шага — не больше:

1. *Суть одной фразой:* Скилл — это когда ты сохраняешь повторяющуюся инструкцию для Claude и запускаешь её одним словом.

2. *Зачем одной фразой:* Чтобы не объяснять одно и то же каждый раз — написал один раз, запускаешь `/команда`.

3. *Пример из жизни:* Каждый раз просишь Claude "проверь мой код на ошибки, посмотри стиль, убедись что тесты есть"? Создаёшь скилл `/проверь` — и всё это происходит одним словом.

После этого спроси: "Хочешь попробуем создать прямо сейчас?"

Технические детали (пути, YAML, frontmatter) — только ПОСЛЕ того как человек сказал "да".

---

### Что такое скилл (технически — для тех кто уже хочет создать)
Скилл — это `.md` файл с инструкциями, который Claude Code следует когда релевантно. Никакого SDK, API или сборки — только Markdown. Создаёшь файл `SKILL.md`, и Claude добавляет его в свой набор инструментов.

**Когда создавать скилл:**
- Когда снова и снова вставляешь одни и те же инструкции в чат
- Когда секция CLAUDE.md выросла в процедуру, а не набор фактов
- Для автоматизации повторяющихся рабочих процессов

### Где хранятся скиллы

| Уровень | Путь | Доступен |
|---------|------|---------|
| Организация | Управляемые настройки | Всем пользователям |
| Личный | `~/.claude/skills/<имя>/SKILL.md` | Вам во всех проектах |
| Проект | `.claude/skills/<имя>/SKILL.md` | Только этот проект |
| Плагин | `<плагин>/skills/<имя>/SKILL.md` | Где включён плагин |

### Создание скилла — пошагово

```bash
# 1. Создать директорию скилла
mkdir -p ~/.claude/skills/мой-скилл

# 2. Создать SKILL.md
nano ~/.claude/skills/мой-скилл/SKILL.md
```

```yaml
# Содержимое SKILL.md:
---
name: мой-скилл
description: Что делает скилл и когда его использовать. Claude читает это чтобы решить когда применять скилл.
disable-model-invocation: false
---

## Инструкции

Здесь пишем что делать Claude когда скилл активирован.
```

```bash
# 3. Протестировать (в Claude Code)
/мой-скилл
# или спросить что-то релевантное - Claude сам применит скилл
```

### Frontmatter поля SKILL.md

| Поле | Обязательно | Описание |
|------|-------------|---------|
| `name` | Нет | Имя скилла (становится /командой). Только строчные буквы, цифры, дефисы. |
| `description` | Рекомендуется | Что делает и когда использовать. Claude читает для авто-активации. |
| `when_to_use` | Нет | Дополнительный контекст когда активировать. |
| `argument-hint` | Нет | Подсказка в автодополнении: `[номер-задачи]` |
| `arguments` | Нет | Именованные позиционные аргументы для `$name` |
| `disable-model-invocation` | Нет | `true` — только ручной вызов. Для деплоя, коммитов. |
| `user-invocable` | Нет | `false` — скрыть из меню `/`. Только Claude вызывает. |
| `allowed-tools` | Нет | Разрешённые инструменты без подтверждения |
| `model` | Нет | Модель для этого скилла |
| `effort` | Нет | Уровень мышления: low/medium/high/xhigh/max |
| `context` | Нет | `fork` — запустить в изолированном субагенте |
| `agent` | Нет | Тип субагента при `context: fork` |
| `hooks` | Нет | Хуки для жизненного цикла скилла |
| `paths` | Нет | Glob-паттерны — активировать только для этих файлов |

### Переменные подстановки в SKILL.md

| Переменная | Описание |
|-----------|---------|
| `$ARGUMENTS` | Все аргументы переданные при вызове |
| `$ARGUMENTS[0]`, `$0` | Первый аргумент |
| `$ARGUMENTS[1]`, `$1` | Второй аргумент |
| `$name` | Именованный аргумент из frontmatter |
| `${CLAUDE_SESSION_ID}` | ID текущей сессии |
| `${CLAUDE_EFFORT}` | Текущий уровень мышления |
| `${CLAUDE_SKILL_DIR}` | Директория скилла |

### Динамический контекст — инлайн команды

```yaml
---
description: Резюме изменений и риски
---

## Текущие изменения
!`git diff HEAD`

## Инструкции
Резюмируй изменения в 2-3 пунктах и укажи риски.
```

Синтаксис `` !`команда` `` — выполняет команду ДО того как Claude видит контент. Вывод подставляется на место плейсхолдера.

### Пример: скилл с аргументами

```yaml
---
name: fix-issue
description: Исправить задачу с GitHub по номеру
disable-model-invocation: true
---

Исправь задачу GitHub #$ARGUMENTS по нашим стандартам:
1. Прочитай описание задачи
2. Пойми требования
3. Реализуй исправление
4. Напиши тесты
5. Создай коммит
```

Вызов: `/fix-issue 123`

## ХУКИ (HOOKS) — ПОДРОБНО

### КАК ОБЪЯСНИТЬ ХУКИ НОВИЧКУ (используй именно это при первом знакомстве)

1. *Суть одной фразой:* Хук — это действие которое происходит автоматически, когда Claude что-то делает — например, перед тем как сохранить файл или после того как выполнил команду.

2. *Зачем одной фразой:* Чтобы не следить за Claude вручную — пусть сам проверяет, уведомляет, или блокирует если что-то пошло не так.

3. *Пример из жизни:* Хочешь чтобы Claude всегда запускал тесты после того как изменил код? Создаёшь хук — и это происходит автоматически, без твоего участия.

После этого спроси: "Хочешь настроим твой первый хук?"

Технические детали — только ПОСЛЕ "да".

---

### Что такое хуки (технически)
Хуки — это команды, которые автоматически выполняются в определённые моменты жизненного цикла Claude Code. Позволяют автоматизировать проверки, уведомления, блокировки.

### Все события хуков (21 событие)

**Уровень сессии:**
- `SessionStart` — при старте/возобновлении сессии
- `Setup` — одноразовая инициализация
- `SessionEnd` — при завершении сессии

**Уровень хода:**
- `UserPromptSubmit` — при отправке запроса
- `UserPromptExpansion` — при раскрытии slash-команды
- `Stop` — когда Claude закончил отвечать
- `StopFailure` — ошибка API

**Уровень вызова инструмента:**
- `PreToolUse` — ДО выполнения инструмента (можно заблокировать)
- `PermissionRequest` — при запросе разрешения
- `PermissionDenied` — когда авторежим отказал
- `PostToolUse` — ПОСЛЕ успешного инструмента
- `PostToolUseFailure` — после ошибки инструмента
- `PostToolBatch` — после пакета параллельных инструментов

**Уровень агентов:**
- `SubagentStart` — при старте субагента
- `SubagentStop` — при завершении субагента
- `TaskCreated` — при создании задачи
- `TaskCompleted` — при завершении задачи
- `TeammateIdle` — когда агент в команде простаивает

**Файлы и конфиг:**
- `FileChanged` — при изменении отслеживаемого файла
- `ConfigChange` — при изменении конфига в сессии
- `CwdChanged` — при смене рабочей директории
- `InstructionsLoaded` — при загрузке CLAUDE.md
- `WorktreeCreate` / `WorktreeRemove` — worktree создан/удалён

**Компактизация:**
- `PreCompact` / `PostCompact` — до/после сжатия контекста

**MCP:**
- `Elicitation` / `ElicitationResult` — MCP запрашивает ввод

**Другие:**
- `Notification` — уведомление Claude Code

### Конфигурация хуков в settings.json

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/check-bash.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
```

### Типы обработчиков хуков

**1. command** — выполнить shell-команду:
```json
{
  "type": "command",
  "command": "путь/к/скрипту.sh",
  "timeout": 600,
  "async": false,
  "once": false,
  "statusMessage": "Проверяю..."
}
```

**2. http** — отправить POST на эндпоинт:
```json
{
  "type": "http",
  "url": "http://localhost:8080/hooks/validate",
  "headers": {"Authorization": "Bearer $TOKEN"},
  "allowedEnvVars": ["TOKEN"],
  "timeout": 30
}
```

**3. prompt** — оценить через LLM:
```json
{
  "type": "prompt",
  "prompt": "Эта команда безопасна? $ARGUMENTS",
  "model": "claude-opus-4-5",
  "timeout": 30
}
```

**4. agent** — запустить субагент:
```json
{
  "type": "agent",
  "prompt": "Проверь это изменение: $ARGUMENTS",
  "timeout": 60
}
```

### Коды выхода хука
| Код | Смысл |
|-----|-------|
| `0` | Успех, разобрать JSON-вывод |
| `2` | Блокирующая ошибка, показать stderr Claude |
| Другой | Некритическая ошибка, продолжить |

### Паттерны матчера

| Значение | Поведение |
|---------|---------|
| `"*"`, `""`, отсутствует | Совпадает всегда |
| Буквы/цифры/`_`/`\|` | Точное совпадение или список: `Bash`, `Edit\|Write` |
| Другие символы | Regex: `^Notebook`, `mcp__.*__write` |

### Пример: блокировать опасные команды

```bash
#!/bin/bash
# ~/.claude/hooks/block-rm.sh
COMMAND=$(jq -r '.tool_input.command' < /dev/stdin)

if echo "$COMMAND" | grep -q 'rm -rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "Опасная команда заблокирована"
    }
  }'
else
  exit 0
fi
```

## MCP СЕРВЕРЫ (Model Context Protocol)

### КАК ОБЪЯСНИТЬ MCP НОВИЧКУ (используй именно это при первом знакомстве)

1. *Суть одной фразой:* MCP — это способ подключить Claude к внешним сервисам: базам данных, Figma, GitHub, Slack и чему угодно ещё.

2. *Зачем одной фразой:* По умолчанию Claude работает только с файлами у тебя на компьютере — MCP даёт ему руки чтобы работать с реальными сервисами.

3. *Пример из жизни:* Подключил MCP к GitHub — и теперь можешь попросить Claude "создай PR" или "покажи открытые задачи" прямо из чата.

После этого спроси: "Хочешь подключим какой-нибудь сервис?"

Технические детали — только ПОСЛЕ "да".

---

### Что такое MCP (технически)
MCP — открытый стандарт для подключения AI-инструментов к внешним сервисам. Через MCP Claude Code получает доступ к базам данных, API, инструментам.

### Добавление серверов

```bash
# HTTP-сервер (рекомендуется для облачных)
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  --header "Authorization: Bearer ВАШ_ТОКЕН"

# stdio-сервер (локальный процесс)
claude mcp add --transport stdio airtable \
  --env AIRTABLE_API_KEY=ВАШ_КЛЮЧ -- npx -y airtable-mcp-server

# SSE-сервер (устаревший, лучше http)
claude mcp add --transport sse asana https://mcp.asana.com/sse
```

### Управление серверами

```bash
# Список серверов
claude mcp list

# Детали сервера
claude mcp get github

# Удалить сервер
claude mcp remove github

# В сессии: статус серверов
/mcp
```

### Области видимости (scopes)

| Область | Хранится | Доступен |
|---------|---------|---------|
| `local` (по умолчанию) | `~/.claude.json` | Только текущий проект |
| `project` | `.mcp.json` в корне проекта | Команда через git |
| `user` | `~/.claude.json` | Все ваши проекты |

```bash
# Добавить с областью user
claude mcp add --scope user --transport http stripe https://mcp.stripe.com

# Добавить с областью project (для команды)
claude mcp add --scope project --transport http paypal https://mcp.paypal.com/mcp
```

## CLAUDE.MD — ПАМЯТЬ ПРОЕКТА

### КАК ОБЪЯСНИТЬ CLAUDE.MD НОВИЧКУ (используй именно это при первом знакомстве)

1. *Суть одной фразой:* CLAUDE.md — это файл который Claude читает каждый раз когда открываешь проект, чтобы знать правила твоего проекта.

2. *Зачем одной фразой:* Без него приходится каждый раз объяснять Claude "это за проект, вот как он устроен, вот что нельзя трогать" — с ним это происходит автоматически.

3. *Пример из жизни:* Как инструкция для нового сотрудника — один раз написал "вот наш проект, вот наши правила", и новый Claude сразу всё знает.

После этого спроси: "Хочешь создадим CLAUDE.md для твоего проекта?"

Технические детали — только ПОСЛЕ "да".

---

### Что такое CLAUDE.md (технически)
Файл с инструкциями который Claude читает при каждом старте сессии. Сохраняет контекст между сессиями.

### Где хранить

| Уровень | Путь | Применяется |
|---------|------|-----------|
| Организация (managed) | `/Library/Application Support/ClaudeCode/CLAUDE.md` (macOS) | Все пользователи |
| Личный | `~/.claude/CLAUDE.md` | Все ваши проекты |
| Проект | `./CLAUDE.md` или `./.claude/CLAUDE.md` | Этот проект + команда |
| Локальный | `./CLAUDE.local.md` (в .gitignore) | Только вы, этот проект |

### Что писать в CLAUDE.md

```markdown
# Проект: Название

## Команды запуска
- Установить: `npm install`
- Запустить dev: `npm run dev`
- Тесты: `npm test`
- Линтер: `npm run lint`

## Архитектура
- Frontend: React + TypeScript в `src/`
- API: FastAPI в `backend/`
- БД: PostgreSQL

## Конвенции кода
- 2 пробела для отступа
- Именование: camelCase для переменных, PascalCase для компонентов
- Всегда добавлять типы TypeScript

## Важные правила
- Никогда не пушить напрямую в main
- Перед коммитом запускать `npm test`
- PR требует 1 апрув
```

### Команда /init

Запусти `/init` чтобы Claude автоматически проанализировал проект и создал CLAUDE.md.

### Автопамять (Auto Memory)

Claude сам сохраняет заметки в `~/.claude/projects/<проект>/memory/`:
- Команды сборки
- Паттерны отладки
- Ваши предпочтения

Управление: `/memory` → toggle auto memory

### Правила по путям (.claude/rules/)

```
.claude/
└── rules/
    ├── testing.md           # Правила для всех файлов
    └── api-rules.md         # Только для src/api/**
```

Файл api-rules.md с frontmatter:
```markdown
---
paths:
  - "src/api/**/*.ts"
---

# Правила API
- Всегда валидировать входные данные
- Использовать стандартный формат ошибок
```

## SETTINGS.JSON — НАСТРОЙКИ

### Расположение файлов

| Файл | Область |
|------|---------|
| `~/.claude/settings.json` | Глобальные настройки пользователя |
| `.claude/settings.json` | Проектные настройки (в git) |
| `.claude/settings.local.json` | Локальные проектные (в .gitignore) |

### Пример полезных настроек

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "model": "claude-sonnet-4-6",
  "effortLevel": "high",
  "alwaysThinkingEnabled": true,
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Read(./.env)"
    ]
  },
  "autoMemoryEnabled": true,
  "editorMode": "normal",
  "language": "russian"
}
```

### Ключевые настройки

| Настройка | Тип | Описание |
|-----------|-----|---------|
| `model` | string | Модель по умолчанию |
| `effortLevel` | string | `low`/`medium`/`high`/`xhigh` |
| `alwaysThinkingEnabled` | bool | Всегда расширенное мышление |
| `language` | string | Язык ответов (напр. "russian") |
| `autoMemoryEnabled` | bool | Авто-память |
| `editorMode` | string | `normal` или `vim` |
| `permissions.allow` | array | Разрешить без подтверждения |
| `permissions.deny` | array | Запретить всегда |
| `permissions.defaultMode` | string | Режим: `default`/`auto`/`plan` |
| `hooks` | object | Хуки жизненного цикла |
| `env` | object | Переменные окружения |
