## СРЕДЫ РАЗРАБОТКИ (IDE)

### VS Code

**Для новичка:** VS Code — это как рабочий стол где лежат все твои файлы проекта, есть подсветка кода, встроенный терминал и расширения на любой вкус. Самый популярный редактор в мире.

**Зачем:** Claude Code работает прямо в терминале VS Code. Ты видишь код и изменения — Claude их делает. Это и есть вайб-кодинг в действии.

**Ключевые расширения для вайб-кодинга:**
- `Claude Code` (официальное) — Claude прямо в боковой панели
- `GitLens` — видишь кто и когда менял каждую строку
- `Thunder Client` — тестируешь API прямо в редакторе, без Postman
- `Docker` — управляешь контейнерами
- `Telegram` — превью Markdown как в Telegram
- `Python` + `Pylance` — автодополнение и проверка типов для Python

**Лайфхаки VS Code:**
- `Ctrl+` ` ` — открыть/закрыть терминал одной кнопкой
- `Ctrl+P` — найти любой файл мгновенно
- `Ctrl+Shift+P` — палитра команд (всё что угодно)
- `Ctrl+D` — выделить следующее совпадение (удобно для переименования)
- Встроенный Git: кнопка Source Control слева — коммиты без терминала

### Cursor

**Для новичка:** Cursor — это VS Code но с AI встроенным везде. Как будто VS Code и Claude Code слились в одну программу.

**Зачем:** Можно выделить кусок кода и прямо в редакторе написать "перепиши это используя async/await" — и Cursor сделает прямо там, не переключаясь в терминал.

**Чем отличается от Claude Code:**
- Claude Code = AI в терминале, работает с проектом целиком
- Cursor = AI в редакторе, работает с конкретным файлом/фрагментом
- Лучше использовать вместе: Cursor для правок файлов, Claude Code для задач по всему проекту

**Ключевые фишки:**
- `Ctrl+K` — редактировать выделенный код через AI
- `Ctrl+L` — чат с AI о текущем файле
- `Tab` — умное автодополнение (лучше чем обычный intellisense)
- `Ctrl+Shift+L` — добавить файлы в контекст чата

**Скачать:** cursor.com (бесплатный план есть)

### Warp (только Mac/Linux)

**Для новичка:** Warp — это терминал (командная строка) с AI внутри. Как обычный терминал, но можно написать по-русски "покажи все файлы изменённые сегодня" — и он сам переведёт в команду.

**Зачем:** Не нужно помнить команды. Описываешь что хочешь — Warp пишет команду за тебя.

**Лайфхаки:**
- `Ctrl+~` — AI-подсказка прямо в командной строке
- Блоки команд — каждая команда и её вывод в отдельном блоке, удобно копировать
- Workflows — сохрани часто используемые команды как шаблоны

**Скачать:** warp.dev

## ГОЛОСОВОЙ ВВОД

### AquaVoice (Mac)

**Для новичка:** AquaVoice — это как умный переводчик с голоса в текст. Говоришь — оно печатает в любом поле, терминале, чате. Только устно, без рук.

**Зачем в вайб-кодинге:** Описываешь задачу голосом прямо в Claude Code. Руки свободны, скорость выше — особенно для длинных промптов.

**Важно:** AquaVoice передаёт голос в текст. Ты всё равно говоришь Claude Code текстом — просто вводишь его голосом, не клавиатурой.

**Горячая клавиша:** Удерживай Right Option (⌥) чтобы говорить, отпусти — текст вставится.

**Лайфхак:** Используй с Cursor или терминалом VS Code. Говоришь задачу → Claude делает → смотришь результат.

**Скачать:** aqua.new

---

### Wispr Flow (Mac/Windows)

**Для новичка:** Аналог AquaVoice, но работает на Windows тоже. Та же идея — голос в текст в любом приложении.

**Скачать:** wisprflow.ai

## БЫСТРОЕ ПРОТОТИПИРОВАНИЕ

### bolt.new

**Для новичка:** bolt.new — это как Claude Code но в браузере, без установки. Описываешь что хочешь сделать — он генерирует готовое приложение которое сразу запускается.

**Зачем:** Хочешь быстро проверить идею — открыл bolt.new, написал "сделай телеграм-бота для записи задач", через 2 минуты есть рабочий прототип.

**Когда использовать:**
- Быстро проверить идею (за 5 минут)
- Сгенерировать стартовый код который потом доделывать в VS Code
- Объяснить кому-то как работает идея

**Лайфхак:** Экспортируй из bolt.new в GitHub, а дальше работай через Claude Code — лучшее из двух миров.

**Адрес:** bolt.new

### v0.dev (от Vercel)

**Для новичка:** v0.dev делает красивые интерфейсы из описания. Написал "сделай дашборд с графиками продаж" — получил готовый React-компонент который можно скопировать.

**Зачем:** Фронтенд — самое сложное для вайб-кодера. v0 решает это за секунды.

**Адрес:** v0.dev

## ДЕПЛОЙ И ХОСТИНГ

### Railway (уже используем)

**Лайфхаки которые не все знают:**
- `railway logs --tail` — логи в реальном времени в терминале
- `railway run python bot.py` — запустить скрипт с переменными Railway локально
- `railway shell` — войти в контейнер как по SSH
- Несколько сервисов в одном проекте — бот + база + redis одной кнопкой
- Автодеплой из GitHub при каждом пуше — включается в настройках

---

### Vercel

**Для новичка:** Vercel — это Railway но специально для веб-сайтов и API. Загружаешь код — через 30 секунд сайт в интернете с HTTPS.

**Зачем:** Если делаешь веб-приложение или API на FastAPI/Next.js — Vercel проще и быстрее Railway.

**Лайфхак:** Vercel даёт бесплатный URL вида `твой-проект.vercel.app` — удобно для тестирования вебхуков Telegram.

---

### Ngrok

**Для новичка:** Ngrok — это как временный туннель из интернета к твоему компьютеру. Telegram не может достучаться до твоего localhost — ngrok даёт временный URL который работает.

**Зачем:** Тестировать вебхуки Telegram прямо на своём компьютере, без деплоя на Railway каждый раз.

```bash
# Установка
brew install ngrok  # Mac

# Запуск туннеля
ngrok http 8000

# Получишь URL вида: https://abc123.ngrok.io
# Его вставляешь как вебхук в Telegram
```

## БАЗЫ ДАННЫХ

### Supabase

**Для новичка:** Supabase — это как Excel таблица в облаке, только умная. Есть интерфейс где видишь все данные, можно делать запросы, и это всё доступно из кода.

**Зачем:** Бесплатная база данных PostgreSQL в облаке + удобный интерфейс + авторизация пользователей из коробки.

**Лайфхак:** Supabase даёт REST API автоматически — можно работать с базой без SQL вообще.

```python
from supabase import create_client
client = create_client(SUPABASE_URL, SUPABASE_KEY)
# Добавить запись
client.table("users").insert({"name": "Олег"}).execute()
# Получить все записи
data = client.table("users").select("*").execute()
```

---

### TablePlus

**Для новичка:** TablePlus — это красивый интерфейс для любой базы данных. Как Excel, только для SQLite, PostgreSQL, MySQL и других.

**Зачем:** Смотреть что реально лежит в базе во время разработки. Незаменимо для отладки.

**Скачать:** tableplus.com (есть бесплатная версия)

## ТЕСТИРОВАНИЕ API

### Thunder Client (расширение VS Code)

**Для новичка:** Thunder Client — это как Postman но прямо в VS Code. Отправляешь запросы к API и видишь ответы не выходя из редактора.

**Зачем:** Тестировать свой API (или чужой) без лишних программ.

**Лайфхак:** Сохраняй коллекции запросов в папке проекта — Claude Code видит их и может помочь с отладкой.

## АВТОМАТИЗАЦИЯ И ПРОДУКТИВНОСТЬ

### Raycast (Mac)

**Для новичка:** Raycast — это как Spotlight (⌘+Space) но в 10 раз мощнее. Запускать приложения, искать файлы, переводить текст, открывать GitHub репо — всё одной горячей клавишей.

**Зачем вайб-кодеру:**
- AI Commands — спрашиваешь что угодно не открывая браузер
- Snippets — сохраняешь часто используемые куски кода/промптов
- Clipboard History — видишь всё что копировал за последние дни

**Расширения для разработки:**
- GitHub — открыть репо одной командой
- Railway — статус деплоев
- Vercel — деплой прямо из Raycast

**Скачать:** raycast.com (бесплатно)

---

### Make (ex-Integromat) и Zapier

**Для новичка:** Make/Zapier — это как конструктор автоматизаций. Соединяешь сервисы без кода: "когда приходит письмо на Gmail → создай задачу в Notion → отправь уведомление в Telegram".

**Зачем:** Если нужна автоматизация между сервисами — Make/Zapier быстрее чем писать код. Claude Code хорошо помогает настраивать Make-сценарии.

## ЛАЙФХАКИ ВАЙБ-КОДИНГА

### Воркфлоу голосового кодинга
1. Открой VS Code с терминалом
2. Запусти `claude` в терминале
3. Включи AquaVoice (Right Option)
4. Говори задачи голосом прямо в Claude Code
5. Смотри как код пишется сам

### Как давать задачи Claude Code эффективно
- **Плохо:** "сделай бота"
- **Хорошо:** "сделай Telegram-бота на aiogram 3, который принимает текст и отвечает через Claude API. Токены из переменных окружения."
- **Контекст решает всё** — чем точнее описание, тем меньше итераций

### Screenshot debugging
Сделал скриншот ошибки → перетащил в Claude Code или отправил боту → объяснение и решение за секунды. Работает лучше чем копировать текст ошибки.

### Итеративный подход
Не пиши весь проект сразу. Итерации:
1. Сделай минимальный рабочий прототип (5 минут)
2. Проверь что работает
3. Добавь следующую фичу
4. Повторяй

### CLAUDE.md как суперсила
Напиши в CLAUDE.md один раз:
```markdown
# Мой проект
Telegram-бот для X. Используем aiogram 3, Python 3.11.

# Правила
- Все тексты на русском
- Токены только из .env
- Деплой на Railway через git push
```
И Claude Code знает контекст в каждой сессии без повторных объяснений.

### .env файл — обязательная практика
Никогда не хардкодь токены в код. Всегда:
```python
import os
TOKEN = os.environ.get("BOT_TOKEN")
```
Файл `.env` локально, переменные Railway в продакшне.

### Быстрый деплой воркфлоу
```bash
git add -A
git commit -m "описание изменений"
git push  # Railway автодеплоит
railway logs  # смотришь результат
```
Весь цикл — 30 секунд.

### Используй Claude Code для документации
После завершения фичи:
```
напиши README для этого проекта
```
Claude прочитает весь код и напишет документацию сам.

### Параллельная разработка с worktrees
Пока Claude делает большую задачу в одном worktree — работаешь в другой ветке. Никаких блокировок.
```bash
claude --worktree  # создаёт изолированную копию
```
